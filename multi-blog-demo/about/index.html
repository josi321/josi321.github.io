About this demo

This site is a working demo of a Hugo-based multi-blog framework with Anthropic-assisted drafting and Cloudflare Pages deployment. Three distinct publications run from one repo, one deploy pipeline.

How it works

  • Hugo content sections. Each blog lives under content/<section>/. Adding a fourth blog is one new directory and one new entry in hugo.yaml. No layout work.
  • Per-section RSS. Each blog gets its own feed at /<section>/index.xml. Readers subscribe to what they care about.
  • Per-section tone profile. params.blogs carries a tone string per section. The drafting helper reads that to keep voice consistent within a blog.
  • Anthropic-assisted drafts. The draft_post.py helper takes a topic, picks the section, prompts Claude with the tone profile, and writes a draft to content/<section>/<slug>.md for human review. Drafts never auto-publish.
  • Cloudflare Pages deploy. Push to GitLab triggers the CF Pages build hook. The site rebuilds and goes live in under 30 seconds. DNS lives at Cloudflare; the GitLab repo can be private.

What this is not

This is not a CMS. There’s no admin UI, no comment system, no user accounts. It’s a framework for a single operator (or small team) running many small publications. If you need a multi-author CMS with role-based publishing, this is the wrong shape.
