This demo shows a working pattern for running multiple distinct blogs out of a single Hugo project. Each blog gets its own section, its own RSS feed, its own tone-of-voice profile for AI-assisted drafting, and its own canonical URL path. One repo, one deploy pipeline, three independent publications.

The three blogs below are illustrative. The framework supports adding more without touching layouts: add a directory under content/, add an entry to params.blogs in hugo.yaml, ship.
